Responsive Breakpoints

Mobile-first approach using Bootstrap-inspired breakpoints. Start with mobile styles, then add responsive overrides.

Breakpoint Min Width CSS Variable Description
XS 0 --breakpoint-xs Mobile phones (portrait)
SM 576px --breakpoint-sm Mobile phones (landscape)
MD 768px --breakpoint-md Tablets (portrait)
LG 992px --breakpoint-lg Tablets (landscape) / Small laptops
XL 1200px --breakpoint-xl Desktops
XXL 1400px --breakpoint-xxl Large desktops
/* Mobile-first approach */ .component { padding: var(--space-4); /* Base mobile styles */ } /* Tablet and up */ @media (min-width: 768px) { .component { padding: var(--space-6); } } /* Desktop and up */ @media (min-width: 1200px) { .component { padding: var(--space-8); } }

Mobile-First Guidelines

Design principles for responsive VS Code-like interface.

Layout Behavior

Component Mobile (XS-SM) Tablet (MD) Desktop (LG+)
Header Full width, hamburger menu Full width, visible menu Full width, all elements visible
Rail Hidden, accessible via hamburger 48px collapsed rail 48px rail, optional 240px sidebar
Content Full width, single column Minus rail width Minus rail/sidebar, multi-column
Wizard Full screen, stacked steps Card-based, scrollable Split view with preview

Touch Targets

Minimum touch target size: 48x48px for mobile devices. Icons in rail and buttons should meet this requirement.
