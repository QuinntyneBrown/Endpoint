# Endpoint.Angular

The Angular code generation module for the Endpoint framework. This library provides generators and utilities for creating Angular components, services, modules, and workspace configurations.

## Overview

Endpoint.Angular enables the generation of Angular frontend code that integrates seamlessly with backend services generated by other Endpoint modules. It provides:

- **Component Generation**: Angular components with templates and styles
- **Service Generation**: Injectable services for API communication
- **Module Scaffolding**: Angular modules with routing
- **Workspace Configuration**: Angular CLI workspace setup
- **JSON Manipulation**: Utilities for working with Angular configuration files

## Project Structure

```
Endpoint.Angular/
├── Artifacts/              # Angular artifact generators
├── Syntax/                 # TypeScript/Angular syntax utilities
├── ConfigureServices.cs    # DI registration
├── JObjectExtensions.cs    # JSON manipulation helpers
└── Usings.cs              # Global using directives
```

## Key Features

### Service Registration

```csharp
using Endpoint.Angular;

services.AddAngularServices();
```

### JSON Configuration

The `JObjectExtensions` class provides utilities for manipulating Angular configuration files (angular.json, package.json, etc.):

```csharp
// Work with Angular workspace configuration
var workspace = JObject.Parse(angularJsonContent);
workspace.AddProject("my-component", projectConfig);
```

### Generated Artifacts

- Angular components (standalone and module-based)
- Services with HttpClient integration
- Modules with routing configuration
- Pipes and directives
- Models and interfaces

## Dependencies

- **Endpoint** (Core) - Foundation library
- **Newtonsoft.Json** - JSON manipulation

## Integration

Endpoint.Angular is typically used through the Endpoint.Cli:

```bash
# Generate an Angular component
endpoint angular component --name CustomerList

# Generate an Angular service
endpoint angular service --name CustomerApi

# Generate an Angular module
endpoint angular module --name Customers --routing
```

Or through the specialized Angular CLI tool (Endpoint.Angular.Cli):

```bash
nd-ng component --name MyComponent
```
